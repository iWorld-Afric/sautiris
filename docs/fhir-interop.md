# FHIR Interoperability

SautiRIS supports FHIR R4/R5 resource generation for interoperability with health information exchanges, EMR systems, and national health registries.

## Supported Resources

| FHIR Resource | SautiRIS Source | Description |
|---------------|----------------|-------------|
| `ServiceRequest` | Radiology Order | Imaging order request |
| `ImagingStudy` | Order + PACS | Imaging study with series/instances |
| `DiagnosticReport` | Radiology Report | Structured radiology report |

## Configuration

```env
SAUTIRIS_ENABLE_FHIR=true
SAUTIRIS_FHIR_BASE_URL=http://fhir-server:8080/fhir
SAUTIRIS_FHIR_AUTH_TOKEN=<bearer_token>
```

## Resource Builders

SautiRIS provides builder functions in `sautiris.integrations.fhir.resources`:

### ServiceRequest

Converts a radiology order to a FHIR ServiceRequest:

```python
from sautiris.integrations.fhir.resources import build_service_request

sr = build_service_request(
    order_id="order-001",
    patient_id="patient-001",
    modality="CT",
    accession_number="ACC-001",
    urgency="STAT",
    procedure_code="71020",
    procedure_description="Chest X-Ray",
    clinical_indication="Persistent cough",
    requesting_physician="Dr. Smith",
)
# sr is a validated fhir.resources.ServiceRequest object
```

### ImagingStudy

Converts order + PACS metadata to a FHIR ImagingStudy:

```python
from sautiris.integrations.fhir.resources import build_imaging_study

study = build_imaging_study(
    order_id="order-001",
    patient_id="patient-001",
    modality="CT",
    accession_number="ACC-001",
    study_instance_uid="1.2.840.113619.2.55.3.2831148180.947.1228761365.192",
    num_series=3,
    num_instances=150,
)
```

### DiagnosticReport

Converts a radiology report to a FHIR DiagnosticReport:

```python
from sautiris.integrations.fhir.resources import build_diagnostic_report

report = build_diagnostic_report(
    report_id="report-001",
    order_id="order-001",
    patient_id="patient-001",
    status="final",
    impression="No acute findings",
    imaging_study_id="study-001",
)
```

## Validation

All resources are validated via the `fhir.resources` library (FHIR R5 conformant). Invalid resources raise `ValidationError` on construction.

```python
# Resources validate on creation
report = build_diagnostic_report(
    report_id="r1",
    order_id="o1",
    patient_id="p1",
)

# Serialize to JSON
json_str = report.model_dump_json()

# Serialize to dict
report_dict = report.model_dump()
```

## HL7v2 Interoperability

SautiRIS also supports HL7v2 messaging for legacy system integration. See the HL7v2 section below.

### Parsing Inbound Messages

```python
from sautiris.integrations.hl7v2.parser import parse_orm_o01, parse_oru_r01

# Parse an inbound ORM^O01 (order message)
order = parse_orm_o01(raw_message)
print(order.placer_order_number, order.patient.patient_id)

# Parse an inbound ORU^R01 (result message)
result = parse_oru_r01(raw_message)
print(result.observations)
```

### Building Outbound Messages

```python
from sautiris.integrations.hl7v2.builder import build_orm_o01, build_oru_r01

# Build an ORM^O01
msg = build_orm_o01({
    "order_control": "NW",
    "placer_order_number": "ORD-001",
    "patient_id": "PAT-001",
    "family_name": "Doe",
    "given_name": "John",
    "procedure_code": "71020",
    "procedure_description": "Chest X-Ray",
})

# Build an ORU^R01
msg = build_oru_r01(
    report_data={
        "placer_order_number": "ORD-001",
        "result_status": "F",
        "patient_id": "PAT-001",
    },
    observations=[
        {"value_type": "TX", "observation_id": "FINDINGS", "value": "Normal chest"},
    ],
)
```

### Round-Trip Fidelity

Messages built with the builder can be parsed back with the parser, preserving all key fields:

```python
built = build_orm_o01({"order_control": "NW", "placer_order_number": "ORD-001"})
parsed = parse_orm_o01(built)
assert parsed.order_control == "NW"
assert parsed.placer_order_number == "ORD-001"
```

## National Registry Integration

For Kenya NHIF and DHIS2 integration, mount SautiRIS within SautiCare which provides the DHIS2 and NHIF modules. FHIR resources generated by SautiRIS can be submitted to national registries via SautiCare's interoperability layer.
